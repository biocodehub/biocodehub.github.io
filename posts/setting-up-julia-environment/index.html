Setting Up Julia Environment

Author: David Feng
Sep 20, 2024

Install Julia

Step 1: Download & Install Julia

Visit the official Julia website and download the installer for your operating system.

Step 2: Install Julia Extension for Visual Studio Code

To use Julia in VSCode, you need to install the Julia extension. Please follow the official website for installation instructions.

Environment Setup in VSCode - Mac Version

Assuming VSCode is already installed, the process for setting up the Julia environment is as follows:

  • Confirm the virtual environment path
  • Create Julia Virtual Environment
  • Install IJulia
  • Add the virtual environment as a Jupyter kernel
  • Confirm the kernel has been successfully added
  • Select the kernel in Jupyter Notebook

Detail explanation

1. Confirm the virtual Environment Path

If your virtual environment is named env_bio and is located in the project directory, the path to the virtual environment might be :

# Input
pwd
# Output
/path/to/your/project/env_bio

2. Create Julia Virtual Environment

In the project directory, start the Julia REPL, enter Pkg mode, and create a new virtual environment :

Start Julia REPL by typing the following in your terminal :

# Input
julia

Once inside Julia REPL, enter Pkg mode by typing:

# Input
]

In Pkg mode, create a new virtual environment:

# Input
activate .

This will create Project.toml and Manifest.toml files in the current directory and activate the virtual environment.

3. Install IJulia

Next, install the required packages for your project. For example, install IJulia and any other project-specific packages :

# Input
add IJulia
add YourOtherPackages

After installation, the virtual environment is ready.

4. Add the virtual environment as a Jupyter kernel

After creating the virtual environment, you need to register the Julia kernel with Jupyter Notebook.

Run the following command in the Julia REPL to install the kernel :

# Input
using IJulia
IJulia.installkernel("XXXX-Kernel")

XXXX-Kernel is the name you want to assign to this kernel. It will appear in the Jupyter Notebook kernel list.

5. Confirm the kernel has been successfully added

You can confirm the kernel has been added to Jupyter by listing all available kernels :

# Input
jupyter kernelspec list
# Output
Available kernels:
  XXXX-Kernel-1.10     /Users/username/Library/Jupyter/kernels/XXXX-Kernel-1.10

6. Select the kernel in Jupyter Notebook

  • Open VSCode and open a Jupyter Notebook file (.ipynb).
  • Click the Kernel dropdown at the top of the Notebook page.
  • Select the newly added ‘XXXX-Kernel’ kernel from the dropdown menu.

Remove Specified Kernel

To remove a Julia virtual environment and its corresponding Jupyter kernel, follow these steps :

1. List all available kernels

First, use the following command to list all installed Jupyter kernels and identify which one you want to remove :

# Input
jupyter kernelspec list
# Output
Available kernels:
  XXXX-Kernel-1.10     /Users/username/Library/Jupyter/kernels/XXXX-Kernel-1.10

2. Remove the Jupyter kernel

After identifying the kernel, proceed to remove it. Start by opening Julia :

# Input
julia

Inside Julia REPL, use the rm function to remove the kernel folder. For example, if the kernel is located in /Users/username/Library/Jupyter/kernels/XXXX-Kernel-1.10, use the following command :

# Input
rm("/Users/username/Library/Jupyter/kernels/XXXX-Kernel-1.10", recursive=true, force=true)

This will remove the kernel folder, effectively unregistering the Julia environment from Jupyter.

To ensure the kernel was successfully removed, run jupyter kernelspec list again :

# Input
jupyter kernelspec list
# Output
Available kernels:
  #(No more  XXXX-Kernel-1.10)
